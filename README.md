# Deal Coin

同じ数字のコインをマージして、より大きな数字を目指すパズルゲーム。広告でよく見るあの手のやつを、依存ライブラリなしの素の HTML / CSS / JavaScript で作った。

プレイ: https://nna774.github.io/deal-coin/

## ルール

### 盤面
- 盤面は複数の**セル**からなり、各セルには **10 個のスロット**がある。
- コインはセルの**手前(トップ)から積む**スタック構造。1 つのセルに異なる数字が積み重なる（混在する）こともある。
- セル数は最初 4 個（`最大コイン + 2`）。最大コインが上がるごとに 1 セル解放され、**最大 12 個**まで増える。

### マージ
- あるセルが **10 個すべて同じ数字 `n`** で埋まると、自動的に **`n+1` のコイン 2 枚**に変化する（そのセルは `[n+1, n+1]` になり 8 スロット空く）。
- 数字は 5ⁿ 換算で価値が保存される（10 × 5ⁿ = 2 × 5ⁿ⁺¹）。

### コインの移動
- セルの**トップから連続する同じ数字のまとまり**を掴んで、別のセルへ移せる。
- 移動先は「**空のセル**」か「**トップが同じ数字のセル**」だけ（違う数字の上には積めない）。
- まとまりが移動先に入りきらない時は、**空いているぶんだけ**動かせる（残りは元のセルに残る）。例: 空きが 2 のセルへ、まとまり 3 個のうち 2 個だけ移す。ちょうど 10 個で埋まればそこでマージが起きる。

### ディール
- 「ディール」ボタンで、いくつかの数字がそれぞれ数枚まとまってセルへ配られる。
- 制約は「**1 回のディールで、1 つのセルに配る数字は 1 種類**」だけ（＝異なる数字は別々のセルへ）。セルの中身は問わず、空セルにも、別の数字が入ったセルの上にも積まれる（移動と違って一致は不要）。
- 配られる数字は「現在の最大コインより下」の範囲からランダム。
- 基本はランダムだが、配った後に**どのセルも動かせなくなる（合法手が無い）ような配りは、空きがある限り避ける**（別の配り方を引き直す）。どう配っても避けられない時はそのまま配る。

### 詰み
- **盤面が全セル埋まる（空きスロットが 0）と詰み**で、それ以上は進めない。「はじめから」でやり直す。混ざったコインを埋めすぎると詰みやすいので、動かして整理しながら進める。

### ゴール
- エンドレス。より大きなコインとハイスコアを目指す。スコア・進行状況・ハイスコアは `localStorage` に自動保存され、リロードで続きから遊べる。

## 遊び方（操作）
- セルをタップすると、トップのまとまりが選択され、移動できる先が緑枠で光る。
- 移動先のセルをタップすると移動。もう一度同じセルをタップで選択解除。
- 手詰まりになったら「ディール」で補充する。
- 「はじめから」は確認のうえ最初からやり直す。
- （デバッグ用）盤面は操作のたびにブラウザの DevTools コンソールへ `max=.. score=.. :: [..] [..]` 形式で出力される。各 `[...]` が 1 セルで、左が底・右がトップ。

## ローカルで動かす
静的ファイルだけなので、`index.html` をブラウザで開くだけ。

```sh
open index.html
# もしくはローカルサーバ経由で
python3 -m http.server
```

## 構成
- `index.html` — 画面の骨組み
- `style.css` — レイアウト・コインの配色・演出（マージ時のグロウ、ディール時の落下）
- `game.js` — ゲームロジック一式（状態・移動・マージ・ディール・描画・保存）

序盤の難易度や配布の傾向は `game.js` 冒頭の定数（`START_MAX_COIN` / `INITIAL_COINS` / `DEAL_COUNT_MAX` / `dealFloorFn` など）で調整できる。
